## Tech + กติกาหลัก

- HTML + CSS + Vanilla JS + Canvas 2D, ES Modules, ไม่มี build/framework/CDN จำเป็น
  (ยกเว้น Google Fonts + มี fallback) — deploy ได้บน GitHub Pages/Netlify
- Canvas รองรับ devicePixelRatio + resize / game loop เดียว rAF + delta time /
  หยุดเมื่อแท็บ hidden (visibilitychange)
- **i18n 3 ภาษา th/en/ja**: ข้อความทุกตัวมาจาก `js/i18n.js` + `js/data/content.js`
  ห้าม hardcode — key ครบ 3 ภาษาเสมอ, สลับจาก HUD ไม่ reload (panel/resume ที่เปิดค้าง
  วาดใหม่ผ่าน `i18n.onChange`), จำใน localStorage, default เดาจาก navigator.language
- ★★ **ลำดับความสำคัญของ input (แก้บั๊ก 2026-07-20 รอบ 12)**
  จอยสติ๊ก → คีย์บอร์ด → จุดหมายจากคลิก · และ **กดปุ่มเดิน/จับจอย = ล้าง `moveGoal` ทันที**
  - ถ้าไม่ล้าง: ปล่อยปุ่มแล้วตัวละคร "เดินต่อเอง" ไปหาจุดที่คลิกไว้ก่อนหน้า
  - ★ **ห้าม `held.clear()` ตอนคลิก** — ปุ่มที่ยังกดค้างอยู่จริงจะหลุดจาก set
    ต้องปล่อย-กดใหม่ถึงจะเดินได้ (อาการ "เดินติด" ที่เจ้าของแจ้ง)
  - เทสด้วย `scratchpad/controls.py`
- ★ **คลิกขวาปิด panel ต้องไม่วนเปิดใหม่** — `#panel` ต้องปิดเฉพาะปุ่มซ้าย (`e.button === 0`)
  ไม่งั้นคลิกขวาบนฉากมืดจะปิด panel แล้ว contextmenu ที่ตามมาไปโดน `onRightClick`
  ซึ่งเปิดวัตถุเดิมซ้ำ = วนไม่รู้จบ · `#panel` มี contextmenu ของตัวเอง (preventDefault + close)
- ★ **คลิกเพื่อเดิน (2026-07-20 รอบ 10 — เจ้าของ: "การควบคุมไม่สมูทเลย")**
  - คลิกซ้ายบนพื้น = ตั้งจุดหมาย เดินไปเอง · **คลิกบนวัตถุ = เดินไปแล้วเปิด panel ให้เลย**
  - **คลิกขวา** = `preventDefault` เมนูบันทึกภาพ + หยุดเดิน + interact กับวัตถุในระยะ
  - มีเอฟเฟกต์วงกระเพื่อมตรงจุดคลิก (`renderer.addClickFx`) = รู้ทันทีว่าคลิกติด
  - ★★ **ไม่มี pathfinding** — `standPoint()` จึงต้องเลือก "ด้านของวัตถุที่ผู้เล่นอยู่ใกล้
    ที่สุด" ไม่ใช่ใต้วัตถุเสมอ ไม่งั้นเดินไปโขกวัตถุแล้วค้าง (เจอตอนเทสจริง) ·
    เดินไม่ขยับ 0.5s = เลิกเดิน แล้วเช็คระยะขอบ AABB ว่าใกล้พอจะเปิด panel ไหม
  - จอสัมผัสไม่ผูกคลิกเดิน (มีจอยสติ๊กอยู่แล้ว) กันแตะแล้วเดินมั่ว
- Desktop: WASD/ลูกศร + E/Enter/Space + Esc / Mobile: จอยลอยตามจุดแตะฝั่งซ้าย
  (analog) + ปุ่ม ✦ ขวาล่างโชว์เมื่อใกล้วัตถุ — ตรวจ touch จาก `pointerType==='touch'`
  ห้ามใช้ user agent / viewport เล็กสุด ~360px
- **reduced-motion**: เอฟเฟกต์เช็ค `prefers-reduced-motion` — เวอร์ชันนิ่งยังสวย +
  ปุ่ม ✨ FULL FX opt-in → `localStorage.fx='full'` + class `fx-full` บน `<html>`
  (CSS ใช้ `html:not(.fx-full)`) ★ เครื่องเจ้าของปิด Windows Animation effects —
  ถ้าเทสแล้ว "ไม่เห็นเอฟเฟกต์" เช็คตรงนี้ก่อน
- particle cap: title 420 / ฝุ่นเท้าในห้อง 90 — ทุกฉากหนักใช้ pre-render sprite +
  `drawImage` เท่านั้น
- Resume Mode = HTML scroll ปกติ (a11y fallback, screen reader อ่านได้) เปิดได้ตั้งแต่
  หน้า Title, z-index 180 ใต้ HUD 200
  - ★ **โครงหน้า (2026-07-20 รอบ 11)**: สรุป 3 บรรทัด → กล่อง "ข้อมูลสำหรับ HR" +
    ปุ่มติดต่อ → กลุ่มหัวข้อตาม `resume.groups` → ท้ายหน้า (วันที่อัปเดต · colophon · เครดิตเพลง)
  - ★★ **`resume.groups` อยู่ใน content.js ไม่ใช่ hardcode ใน resume.js** เพราะ
    **ฉบับ ja ใช้โครง 職務経歴書 ของญี่ปุ่นจริง** (職務経歴 → 実績 → 活かせる経験・知識・技術
    → 学歴 → コンタクト) ซึ่งลำดับและหัวข้อไม่เหมือน th/en — นี่คือจุดขายกับ HR ญี่ปุ่น
    ห้ามยุบกลับเป็นลิสต์เดียวใช้ร่วมกัน
  - มี `@media print` แล้ว (HR ชอบสั่ง Print → PDF) — ธีมจอเป็นครามเข้ม ถ้าไม่มีบล็อกนี้
    จะพิมพ์ออกมาเป็นกระดาษดำ · บล็อกนี้บังคับโชว์ Resume Mode แม้ยังไม่เปิดโหมด
- `window.__game` = debug hook สำหรับ automated test —
  `player · panels · input · renderer · map · objects · resume · hover · inGame`

### ★ ความละเอียดภาพ (แก้ "หน้าต่างภาพแตก" 2026-07-20)

สูตรที่ต้องจำ: **world→device = camera.scale × devicePixelRatio**
· `camera.scale = max(screenH/900, screenW/1500)` clamp [0.5, **1.6**] · dpr clamp **2**
· **worst case = 3.2×** — ภาพที่อบไว้ต้อง ≥ ค่านี้ ไม่งั้นถูกขยาย = แตก

วัดจริงบนจอ 1920×1080 dpr2: scale 1.28 × 2 = **2.56×** แต่เดิมอบไว้ 2× → ขยาย 1.28 เท่า
(หลักฐานชัด: ป้ายชื่อที่ JS วาดสดคมกริบ แต่หน้าต่างที่อบมาฟุ้ง อยู่ในภาพเดียวกัน)

วิธีแก้ที่ใช้ = **A+B พร้อมกัน** (รวมแล้วพิกเซลน้อยกว่าอบทั้งห้องที่ 4× ราว 35%):
- `python tools/gen_room_v2.py` → ห้องทั้งใบ **3×** (4800×3000, 292KB)
- `python tools/gen_room_v2.py wall` → **แถบผนังเหนือ 4×** (6400×496, 86KB, RGBA)
  วาดทับภาพฐานใน `renderer.draw()` สูง `WALL_HI_H = 124` world px
  ★ ค่านี้ต้องตรงกับ `BAND_H` ในตัวเจน · alpha ไล่จางที่ขอบล่าง = ไม่เห็นรอยต่อ
- ★ **หมายเหตุสำคัญ**: งาน 3×/4× นี้ **ไม่ใช่** ตัวแก้อาการ "มัว" ที่เจ้าของเห็น
  (จอเจ้าของ dpr 1 → world→device = 0.99× คือภาพถูก *ย่อ* อยู่แล้ว) ตัวการจริงคือ
  แถบเบลอบนของ tilt-shift · แต่ยังเก็บไว้เพราะช่วยจริงบนจอ dpr ≥ 1.5
  (Windows scaling 150%, MacBook, จอ 4K) ซึ่ง HR ใช้กันเยอะ
- ★ วิธีแยกตัวแปรเวลาเจอ "ภาพมัว": `python scratchpad/blur_diag.py`

## Performance (งบ 60fps = 16.7ms/เฟรม)

วัดจริงด้วย `python scratchpad/perf.py` → **renderer.draw = 0.8ms median** (HD-2D เปิด)
★ วัด ms/เฟรมเสมอ **อย่าวัด FPS** — rAF ใน headless ถูก throttle ตัวเลขเชื่อไม่ได้
หลักการที่ทำไว้ ห้ามถอย:
- **ห้ามใช้ `ctx.shadowBlur` ในลูปวาดต่อเฟรม** — Canvas2D shadow = blur pass ต่อ 1

- gradient ทุกอันต้อง pre-render (`buildFxSprites`, `poolFor`, `glowFor`) — ห้าม
  `createRadialGradient`/`createLinearGradient` ในเฟรม
- ห้ามสร้าง object/array ใหม่ทุกเฟรม — `main.js` ใช้ `drawState` ตัวเดิม,
  renderer ใช้ `this._dr` + comparator `sortByY` ถาวร
- `i18n.t()` cache ไว้ (prompt คำนวณใหม่เฉพาะตอนสลับภาษา/touch)

## ★★ กับดัก CSS/DOM ที่เจอในโปรเจกต์นี้

- `#panel-body p { color: หมึกเข้ม }` (id selector) ชนะ class ล้วนเสมอ — กล่องพื้นเข้ม
  ใน panel วาชิต้องเขียนสีด้วย `#panel-body .xxx` นำหน้า (ดูหัวข้อธีม)
- **keyframe ที่ `animation-fill-mode: both` จะล็อกค่าไว้ ทำให้ class อื่นสั่งทับไม่ได้**
  เช่น `#hint` มี `animation: hint-in ... both` แล้วสั่ง `.gone { opacity: 0 }` ไม่สำเร็จ
  ต้องใส่ `animation: none` ใน `.gone` ด้วย (อาการ: ป้ายวิธีเล่นลอยทับ panel ไม่ยอมหาย)
- THEME v2 ตั้ง `img.panel-hero { aspect-ratio: 16/9; max-height: none }` — hero ที่เป็น
  การ์ดโลโก้เลยสูง ~380px ทั้งที่ให้ข้อมูลศูนย์ · override ด้วย `#panel-body img.panel-hero`
  (`aspect-ratio: auto` + `object-fit: contain`) ไม่งั้นเปิด panel มาเจอโลโก้เต็มจอ
